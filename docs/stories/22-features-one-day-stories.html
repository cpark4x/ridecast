Replicable pattern
This pattern is replicable.
The 6 rules are the diff between Phase 1 and Phase 2. Start with one session.
The 6 Spec Quality Rules
  • 1
    Quote current code in the spec. Don't describe it — paste the function. The machine can't refactor what it can't see.
  • 2
    Show the vi.mock() pattern explicitly. Pattern ambiguity creates test failures that create extra sessions.
  • 3
    Scope each spec to ≤5 files. 5-file scopes ship. 15-file scopes get stuck.
  • 4
    Infra specs need human setup steps. List API keys, tenant IDs, cloud resources as pre-conditions. Don't expect the machine to provision.
  • 5
    Scope section is mandatory. Every spec ends with "In scope: X. Out of scope: Y." Without it, the machine optimistically extends.
  • 6
    Add a scope guard to session instructions. working-session-instructions.md must say "do not build anything not in the spec." Belt and suspenders.
