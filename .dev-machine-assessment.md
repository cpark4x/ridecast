## Repository overview (what this project is)

Ridecast2 is a Next.js (App Router) + TypeScript project that turns uploaded/linked content (PDF/EPUB/TXT/URL) into an AI-compressed script (Claude) and then generates TTS audio (OpenAI). It uses Prisma + PostgreSQL for persistence and Playwright for E2E proof scenarios.

Key tech:
- Next.js 16.1.x, React 19, TypeScript
- TailwindCSS 4
- Prisma 7 + PostgreSQL
- Vitest + Testing Library (unit/integration)
- Playwright (E2E)

## Setup (expected on a new dev machine)

### 1) Install dependencies

```bash
npm install
```

### 2) Start a local PostgreSQL (required for DB-backed tests and most integration flows)

The implementation plan expects a local DB at:

```
postgresql://postgres:ridecast@localhost:5432/ridecast
```

One-liner via Docker (from the project docs):

```bash
docker run --name ridecast-db \
  -e POSTGRES_PASSWORD=ridecast \
  -e POSTGRES_DB=ridecast \
  -p 5432:5432 \
  -d postgres:16
```

Then run Prisma migration / generation steps (script names may vary by repo):

```bash
# examples (adjust to actual scripts)
npm run db:migrate
# or
npx prisma migrate dev
npx prisma generate
```

### 3) Runtime API keys (not required for unit tests if mocks are used)

At runtime, the app integrates with:
- Anthropic (Claude)
- OpenAI TTS

You will likely need environment variables such as:
- `ANTHROPIC_API_KEY`
- `OPENAI_API_KEY`

(Tests are intended to mock AI calls; production/dev runtime will need real keys.)

## Blocking issues (ordered by impact)

### P1 — E2E Pattern A: Ambiguous `getByText('Library')` selector

**Specs affected:**
- `e2e/commute-flow.spec.ts:11`
- `e2e/offline-listening.spec.ts:8`
- `e2e/quick-relisten.spec.ts:8`

**Root cause:** `getByText('Library')` matches 3+ DOM elements under Playwright's strict mode, making the selector non-deterministic.

**Fix:** Replace with semantic role-based query:
```ts
// Instead of:
getByText('Library')

// Use:
getByRole('button', { name: 'Library' })
// or:
getByRole('tab', { name: 'Library' })

// Or add data-testid:
getByTestId('library-nav-button')
```

### P1 — E2E Pattern B: Missing "Target Duration" element

**Specs affected:**
- `e2e/pdf-commute.spec.ts:19`
- `e2e/article-discussion.spec.ts:14`

**Symptom:** `getByText('Target Duration')` not found after file upload.

**Possible causes:**
1. webServer config points to incorrect port/URL
2. UI flow missing or changed (element not rendered in upload confirmation)
3. Text/copy mismatch between spec and implementation

**Fix:** 
- Verify `playwright.config.ts` webServer config matches running app (port, baseURL)
- Confirm "Target Duration" label renders in upload flow UI
- If UI changed, update spec to match current flow
